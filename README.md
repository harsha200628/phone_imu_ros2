# 📱 Phone IMU ROS 2 Server (Ubuntu 24.04 & ROS 2 Jazzy)

A professional ROS 2 system that bridges real-time IMU data from a mobile phone into a ROS 2 workspace. Built and tested for **Ubuntu 24.04 (Noble Nightingale)** and **ROS 2 Jazzy Jalisco**.

---

## ⚡ Quick Start (Mini Steps)

### 1. Build
```bash
# Clone and build
git clone https://github.com/harsha200628/phone_imu_ros2.git~/phone_imu_ws/src
cd ~/phone_imu_ws
colcon build --packages-select phone_imu_ws_server
source install/setup.bash
```

### 2. Launch
```bash
ros2 launch phone_imu_ws_server visualize.launch.py
```

### 3. Setup Phone (Chrome Flags)
1.  On phone Chrome: `chrome://flags/#unsafely-treat-insecure-origin-as-secure`
2.  Add: `http://<YOUR_COMPUTER_IP>:8080`
3.  Set to **Enabled** & Relaunch.
4.  Open `http://<YOUR_COMPUTER_IP>:8080` on phone and tap **"Initiate Link"**.

## 📄 Project Report & Documentation

### 🚀 Overview
This package enables any smartphone (Android/iOS) to serve as a wireless IMU sensor. It is designed for low-latency robotics applications where mobile device motion is used to control virtual or physical robots.

### 🏗️ Architecture
The system uses a **Dual-Threaded Async/MW** architecture:
- **Main Thread (AsyncIO)**: Runs two servers concurrently:
  - **HTTP Server (Port 8080)**: Serves the web interface.
  - **WebSocket Server (Port 8765)**: Receives high-frequency JSON motion data.
- **Worker Thread (RCLPY)**: Handles all ROS 2 middleware operations (`spin()`), ensuring that network bottlenecks don't freeze ROS 2 publishing or logging.

### 📁 File Structure & Workflow
| File | Role | Description |
| :--- | :--- | :--- |
| `imu_server.py` | **The Core** | Multi-threaded Python server. Handles WebSocket parsing -> ROS 2 message conversion. |
| `phone_imu.html` | **The Dashboard** | A premium Glassmorphism UI that uses Web APIs to capture phone orientation. |
| `phone.urdf` | **Visual Model** | A vibrant red, double-sized smartphone model for easy tracking in 3D. |
| `visualize.launch.py`| **The Orchestrator**| Launches: Server node + URDF State Publisher + RViz2. |
| `imu_server.launch.py` | **Server Only** | Starts only the bridge without visualization. |

### 📡 ROS 2 API
- **Topic**: `/phone_imu` (`sensor_msgs/Imu`)
  - Publishes: Quaternions (Orientation), Linear Acceleration, and Angular Velocity.
- **Topic**: `/tf` (`tf2_msgs/TFMessage`)
  - Broadcasts dynamic transform: `odom` -> `base_link`.
- **Frame ID**: `phone_imu_link` (Defined in URDF).

### 📐 Physical Enhancements

## 🛠️ Detailed Setup & Troubleshooting

### Enabling Sensors on Insecure Contexts (HTTP)
Browsers strictly block sensor access on HTTP. Since ROS 2 servers often run locally without SSL, you **must** whitelist your computer's IP:
1.  **Identify IP**: Run the launch file; it logs your local IP (e.g., `192.168.1.5`).
2.  **Phone Config**: In Chrome flags, enter the full URL precisely: `http://192.168.1.5:8080`.
3.  **Permissions**: Once the board is whitelisted, you still need to tap **"Initiate Link"** to trigger the OS-level "Accept" dialog for motion sensors.

### System Requirements
- **OS**: Ubuntu 24.04 LTS.
- **ROS 2**: Jazzy Jalisco.
- **Python Dependencies**: `websockets` (`pip install websockets`).
